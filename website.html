Installation

Quick and easy setup to get you started

Homebrew Installation (Recommended for macOS/Linux)

brew tap harsh-2002/homebrew-sona && brew install sona

Quick Setup (Alternative)

curl -fsSL https://raw.githubusercontent.com/Harsh-2002/Sona/main/install.sh | sudo sh

Manual Installation

Download binary

Choose your platform from the download section above

Make executable

chmod +x sona-linux-amd64

Install system-wide

sudo mv sona-linux-amd64 /usr/local/bin/sona

Usage

How to use Sona for your transcription needs

Basic Commands

Set your API key:

sona config set api_key "your_api_key_here"

Check system status:

sona status

Install dependencies:

sona install

Transcription Commands

Transcribe local audio file:

sona transcribe audio.mp3

Transcribe YouTube video:

sona transcribe "https://youtube.com/watch?v=VIDEO_ID"

Transcribe with custom output path:

sona transcribe audio.mp3 --output /path/to/output

Transcribe with specific speech model:

sona transcribe audio.mp3 --model slam-1

Interactive Mode

Start guided interactive mode:

sona

Or explicitly start interactive mode:

sona interactive

Configuration Commands

View current configuration:

sona config show

Set output directory:

sona config set output.default_path "/path/to/transcripts"

Get help for any command:

sona --help

Updating Sona

If installed via Homebrew:

brew upgrade sona

If installed via installer script:

sudo ./install.sh

Troubleshooting

Common issues and how to solve them

API Key Issues

Problem: "API key not configured" or "Authentication failed"

Solution: Set your API key using sona config set api_key "your_key" or check the key is valid in your AssemblyAI dashboard.

YouTube Download Failures

Problem: "yt-dlp not found" or download errors

Solution: Run sona install to install dependencies. If it fails, run sona status to check system status and ensure you have curl/wget available.

Audio Conversion Errors

Problem: "FFmpeg not found" or "transcoding failed"

Solution: Run sona install to install dependencies. Check sona status to verify FFmpeg availability. Ensure audio files are valid and not corrupted.

FFmpeg Extraction Errors

Problem: "xz utility not found" or "failed to extract FFmpeg archive"

Solution: Install xz-utils: sudo apt-get install xz-utils (Ubuntu/Debian) or sudo yum install xz (CentOS/RHEL). This is required for extracting FFmpeg archives on some Linux distributions.

File Permission Issues

Problem: "Permission denied" or "cannot create file"

Solution: Ensure output directory is writable. Run chmod 755 /path/to/output or use sudo if installing system-wide.

Network Timeouts

Problem: "Request timeout" or "connection failed"

Solution: Check internet connection, firewall settings, and try again. AssemblyAI API might be experiencing high traffic.

Unsupported Audio Formats

Problem: "Unsupported format" or "cannot process file"

Solution: Convert audio to common formats (MP3, WAV, M4A) first. The app will attempt automatic conversion using FFmpeg.

Configuration File Issues

Problem: "Config file not found" or settings not saved

Solution: Run sona config show to verify configuration. Check permissions on ~/.sona/ directory.

Memory/Storage Issues

Problem: "Out of memory" or "disk space full"

Solution: Ensure sufficient disk space for audio files and transcripts. Large video files may require significant temporary storage.

Still Having Issues?

Run sona status to check your system configuration and dependency status. If dependencies are missing, run sona install to install them.

sona status
sona install
